# 產品需求文檔（PRD）
## 金憶夜市 - 長者認知訓練與照護平台

**版本**: v0.0.3  
**更新日期**: 2026年4月9日  
**負責團隊**: KIM-I 開發團隊

---

## 1. 產品概述

### 1.1 產品名稱
**金憶** (KIM-I App)

### 1.2 產品願景
開發一個兼具懷舊元素與認知訓練的互動遊戲平台，為長者提供趣味性的心智活動，同時便於照護者監督和管理長者的認知健康狀況。

### 1.3 產品定位
- **目標用戶**: 健康長者及其家庭照護者
- **應用場景**: 家庭日常、老年活動中心、社區關懷機構
- **核心價值**: 結合懷舊場景、認知遊戲、數據追蹤，提升長者生活質量

### 1.4 主要特色
- 🎮 多款互動遊戲（烹飪、購物、音樂、益智等）
- 👴 友善的中文介面，操作簡單直覺
- 📊 照護者儀表板，監測長者活動與進展
- 🎵 懷舊元素與動畫設計，提升使用體驗
- 📱 響應式設計，支援多種裝置
- 👥 多人遊戲模式，支援同樂與輕競賽

## 2. 核心功能需求

### 2.1 遊戲模組

#### 2.1.1 烹飪遊戲 (CookingGame)
**功能描述**: 模擬經典烹飪場景，讓長者進行食材辨識與烹飪步驟記憶  
**目標**: 訓練認知記憶、手眼協調、決策能力  
**互動元素**:
- 食材選擇與組合
- 烹飪步驟序列記憶
- 成品展示與評分

#### 2.1.2 購物遊戲 (ShoppingGame)
**功能描述**: 懷舊夜市購物體驗，訓練預算管理與商品辨識  
**目標**: 認知功能、數學計算、決策判斷  
**互動元素**:
- 虛擬貨幣使用
- 商品選擇與購買
- 預算管理反饋

#### 2.1.3 音樂節奏遊戲 (MusicRhythmGame)
**功能描述**: 跟隨音樂節奏進行互動，結合懷舊樂曲  
**目標**: 聽覺認知、節奏感、反應速度  
**互動元素**:
- 節奏提示與視覺反饋
- 經典樂曲播放
- 成績排名

#### 2.1.4 謎題遊戲 (RiddleGame)
**功能描述**: 經典謎語與腦筋急轉彎  
**目標**: 邏輯思維、語言理解、問題解決  
**互動元素**:
- 謎語呈現與提示
- 答案輸入與驗證
- 難度分級

#### 2.1.5 益智遊戲 (PuzzleGame)
**功能描述**: 拼圖、方塊配對等益智挑戰  
**目標**: 視覺空間認知、邏輯推理、手眼協調  
**互動元素**:
- 拼圖難度調整
- 時間限制挑戰
- 進度追蹤

#### 2.1.6 木偶遊戲 (PuppetGame)
**功能描述**: 互動式故事與角色互動  
**目標**: 敘述理解、創意表達、記憶鞏固  
**互動元素**:
- 故事情節分支
- 角色對話互動
- 故事回顧

#### 2.1.7 長者遊戲系列 (ElderlyGame)
**功能描述**: 專為長者設計的進階認知訓練  
**目標**: 綜合認知能力提升  
**互動元素**:
- 自訂難度設置
- 進階挑戰模式
- 個人成就系統

### 2.2 遊戲菜單 (GameMenu)
**功能描述**: 遊戲首頁與導航  
**主要特性**:
- 遊戲網格展示（3×2 佈局）
- 遊戲狀態指示（可用/開發中）
- 快速導航與友善提示
- 視覺反饋動畫

### 2.3 照護者儀表板 (CaregiverDashboard)
**功能描述**: 監測長者活動、進度與健康指標  
**核心功能**:
- 📈 遊戲進展統計圖表
- 👤 長者個人檔案管理
- 📊 認知能力評分與趨勢
- 🧠 認知面向雷達拆解（記憶、注意力、反應、執行、視覺）
- 🎯 目標設定與追蹤
- 🔔 異常警報與提醒
- 📅 活動日誌與歷史記錄

#### 2.3.1 認知面向雷達拆解
**功能描述**: 將遊戲成績映射到五大認知面向，並在儀表板顯示每個面向的當前分數、基準線與趨勢狀態。  
**映射規則（MVP）**:
- 記憶力：購物遊戲、謎題遊戲
- 注意力：布袋戲遊戲
- 反應力：音樂節奏遊戲
- 執行力：烹飪遊戲
- 視覺空間：拼圖遊戲

**趨勢判定（MVP）**:
- 進步：當前分數較基準線 +5 分以上
- 卡住：當前分數落在基準線 ±4 分
- 下滑：當前分數較基準線 -5 分以下

**驗收標準**:
- [ ] 儀表板可顯示五大面向拆解卡片與雷達圖
- [ ] 每個面向可顯示「當前值 / 基準值 / 趨勢」
- [ ] 可一眼看出「哪個能力進步、哪個能力卡住」
- [ ] 當無遊玩資料時，使用基準線顯示且不報錯

### 2.4 路由與導航 (Router)
**頁面結構**:
- `/` - 遊戲菜單首頁
- `/game/:type` - 各類遊戲頁面
- `/dashboard` - 照護者儀表板
- `/profile/:id` - 長者檔案頁面
- `/multiplayer` - 多人模式大廳
- `/multiplayer/room/:roomId` - 多人房間
- `/multiplayer/result/:sessionId` - 多人結算頁

### 2.5 狀態管理 (GameStore - Pinia)
**管理項目**:
- 使用者認證與檔案信息
- 遊戲進度與統計數據
- 設定偏好與主題選項
- 長短期目標與成就系統

### 2.6 多人遊戲模式 (Multiplayer Mode)
**功能描述**: 提供長者與家人、照護者、同儕共同遊玩的即時多人模式，提升互動動機與訓練持續性。  
**適用遊戲（第一階段）**: 音樂節奏遊戲、謎題遊戲  
**遊玩形式**:
- 合作模式：共同完成目標分數或關卡
- 輕競賽模式：同場比拼分數與穩定度
- 陪玩模式：照護者加入引導，不計入正式排名

**核心需求**:
- 建立房間：房主可建立公開或私人房間
- 邀請機制：透過 6 碼房號或邀請連結加入
- 人數限制：每房 2-4 人（MVP）
- 準備機制：所有玩家 Ready 後開始倒數
- 同步機制：伺服器為權威時間軸，客戶端做插值顯示
- 計分機制：個人成績與隊伍成績並行
- 結算機制：顯示個人表現、團隊結果、鼓勵回饋

**無障礙與長者友善要求**:
- 大字模式下仍可顯示玩家名單與即時分數
- 低壓力文案：以鼓勵語替代強競技語句
- 操作步驟上限：加入房間流程不超過3步

**異常與容錯**:
- 斷線重連：30秒內可回到原房間與進行中對局
- 中途離場：保留已完成數據，並標記為未完賽
- 高延遲保護：延遲超過閾值時自動降低動畫同步精度，確保操作可用

## 3. 非功能需求

### 3.1 性能需求
- **頁面加載時間**: < 2秒
- **動畫幀率**: 60 FPS
- **遊戲響應延遲**: < 100ms
- **支援最小配置**: 
  - 處理器: 双核心 1.5GHz 或以上
  - 記憶體: 2GB 或以上
  - 網絡: 2Mbps 以上

### 3.2 安全性需求
- ✅ 使用者數據加密存儲
- ✅ 身份驗證與授權機制
- ✅ 隱私政策遵循（GDPR、個資法）
- ✅ 定期安全審計

### 3.3 易用性需求
- **字體大小**: 最小 16px（易於閱讀）
- **色彩對比度**: 達到 WCAG AA 標準
- **操作流程**: 不超過3次點擊完成主要任務
- **語言**: 繁體中文、粵語（計畫支援）

### 3.4 相容性需求
- 🖥️ **桌面**: Chrome、Edge、Firefox（最新版本）
- 📱 **行動**: iOS Safari、Android Chrome
- 📐 **螢幕尺寸**: 1024×768 至 4K 分辨率

### 3.5 可靠性需求
- **可用性**: 99.5% 正常運行時間
- **故障恢復**: RTO < 1小時
- **數據備份**: 每日自動備份
- **測試覆蓋率**: > 80%

### 5.1 長者使用者
**故事**: 作為一名 70 歲的退休人士，我希望能通過趣味遊戲度過閒暇時光，同時保持心智敏銳。

**驗收標準**:
- [ ] 我能快速找到並打開一款遊戲

- [ ] 遊戲有趣且不會太複雜

- [ ] 系統會通知異常活動
- [ ] 數據安全且只有授權人員能訪問

### 5.3 醫療人員使用者
**故事**: 作為一名治療師，我想評估患者的認知改善，並根據數據調整訓練計畫。

**驗收標準**:
- [ ] 我能導出詳細的認知評估報告
- [ ] 數據足夠詳細以支持臨床決策
